# agentsec — мультиагентный анализ кода на уязвимости

MVP мультиагентной системы для пентест-анализа кодовых баз. Построено на LangGraph.

## Архитектура

```
Пользователь
    │  задача + путь к репо
    ▼
Оркестратор ──── ask_user (уточнения у пользователя)
    │
    ├── разведка ──► миньоны: explore_codebase, read_docs
    │
    └── делегирование ──► специалисты по классам уязвимостей:
            • analyze_injections     — SQLi, command, path traversal, SSRF, XSS, десериализация
            • analyze_secrets_crypto — секреты в коде, слабая криптография
            • analyze_authnz         — аутентификация и авторизация, IDOR, access control
                    │
                    └── каждый специалист тоже вызывает миньонов
    │
    ▼
Консолидированный markdown-отчёт
```

**Уровни агентов:**
- **Оркестратор** — принимает задачу, ведёт разведку, задаёт вопросы пользователю,
  делегирует специалистам, собирает итоговый отчёт. Только чтение, файлы не правит.
- **Специалисты** — каждый работает по своему классу уязвимостей. Инструменты:
  чтение/запись/редактирование файлов, grep/glob/ls, вызов миньонов. Запись —
  для написания тестов и PoC, с подтверждением пользователя.
- **Миньоны** — узкие сабагенты. `explore_codebase` (разведка кодовой базы),
  `read_docs` (чтение документации). Только чтение.

Каждый агент — самостоятельный ReAct-граф LangGraph; вызов сабагента реализован
как инструмент-обёртка над `.invoke()` его графа.

## Установка

```bash
pip install -r requirements.txt
cp .env.example .env      # и заполни ключ/эндпоинт/модель
```

LLM — любой OpenAI-совместимый бэкенд (OpenAI, OpenRouter, Ollama, vLLM).
Настраивается через `.env`: `OPENAI_BASE_URL`, `OPENAI_API_KEY`, `MODEL`.

## Запуск

```bash
# на встроенном уязвимом демо-приложении
python main.py --repo ./examples/vulnerable_app --task "Проведи аудит безопасности"

# на своём репозитории
python main.py --repo /path/to/target --task "Найди уязвимости в обработке ввода"
```

Флаг `--yes` — авто-подтверждение записи файлов (для неинтерактивных прогонов).

`examples/vulnerable_app/` — намеренно уязвимое приложение (SQLi, command
injection, path traversal, IDOR, хардкод-секреты, MD5) для проверки системы.

## Структура

```
main.py                      CLI-точка входа
agentsec/
  config.py                  глобальная конфигурация рантайма
  llm.py                     универсальная фабрика LLM (OpenAI-совместимый API)
  prompts.py                 системные промпты всех агентов
  tools/
    filesystem.py            read/write/edit/ls/glob/grep (в пределах корня анализа)
    interaction.py           ask_user
  agents/
    minions.py               миньоны explore_codebase, read_docs
    specialists.py           специалисты по классам уязвимостей
    orchestrator.py          оркестратор
```

## Roadmap — отложено за MVP

Сознательно не вошло в MVP, запланировано:

- **Параллельный fan-out специалистов** + общая память/blackboard между ними
  (сейчас оркестратор вызывает специалистов последовательно).
- **Агент-валидатор («скептик»)** — отсев false positives, проверка достижимости
  sink, простановка CVSS.
- **Детерминистические сканеры через MCP** — semgrep, SCA (osv-scanner), поиск
  секретов (gitleaks). LLM-агенты на триаже, сканеры на покрытии.
- **Песочница исполнения** — изолированный контейнер для запуска цели и PoC.
- **Защита от prompt injection** — анализируемый код как недоверенные данные.
- **Структурированная схема находки** (JSON: CWE/CVSS/data-flow/confidence) вместо
  markdown — для машинной дедупликации и трекинга.
- **Трекинг покрытия** — чеклист модулей/эндпоинтов с отметкой «проанализировано».
- **Больше миньонов** — web/CVE-поиск по зависимостям, callgraph/dataflow.
- **Recon-фаза до делегирования** и приоритизация от attack surface.
